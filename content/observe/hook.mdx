# useObserver hook

> The hook is available only with `mobx-react-lite` library or `mobx-react@6`

```ts
useObserver<T>(fn: () => T, baseComponentName = "observed", options?: IUseObserverOptions): T

interface IUseObserverOptions {
    // optional custom hook that should make a component re-render (or not) when changes are detected
    useForceUpdate: () => () => void
}
```

Low level implementation used internally by [observer HOC](/observer-hoc) and [Observer component](/observer-component).

It allows you to use an observer like behaviour, but still allowing you to optimize the component in any way you want (e.g. using memo with a custom areEqual, using forwardRef, etc.) and to declare exactly the part that is observed (the render phase).

One good thing about this is that if any hook changes an observable for some reason then the component won't rerender twice unnecessarily.

```tsx
import { useObserver, useLocalStore } from "mobx-react-lite"

function Person() {
    const person = useLocalStore(() => ({ name: "John" }))
    return useObserver(() => (
        <div>
            {person.name}
            <button onClick={() => (person.name = "Mike")}>No! I am Mike</button>
        </div>
    ))
}
```

Despite using a `useObservable` hook in the middle of the component, it will re-render a whole component on change of the observable. If you want to micro-manage renders, feel free to use [`<Observer />`](/observer-component) or `useForceUpdate` option (for advanced users).
